# نصب و اجرای اولیه

این صفحه دو مسیر پشتیبانی‌شده برای شروع کار با RayLimit را توضیح می‌دهد:

1. نصب از بستهٔ انتشار
2. اجرا از سورس یا بیلد محلی

اگر می‌خواهید RayLimit را مثل یک ابزار معمولی روی میزبان نصب کنید، مسیر بستهٔ انتشار مناسب‌تر است. اگر می‌خواهید بدون دریافت باینری از پیش‌ساخته، ابزار را از روی سورس اجرا کنید، مسیر سورس یا بیلد محلی را انتخاب کنید.

## پیش‌نیازها

RayLimit روی میزبان لینوکسی بیشترین کاربرد را دارد وقتی این موارد در دسترس باشند:

- ابزار `tc` در `PATH`
- حداقل یک runtime از Xray که بتوان آن را کشف و بررسی کرد
- دسترسی root برای اجرای واقعی

## مسیر ۱: نصب از بستهٔ انتشار

```bash
tar -xzf raylimit_v0.1.0-beta_linux_amd64.tar.gz
cd raylimit_v0.1.0-beta_linux_amd64
sudo ./scripts/install.sh
```

این مسیر فایل‌ها را در این مکان‌ها قرار می‌دهد:

- باینری: `/usr/local/bin/raylimit`
- اسکریپت‌ها و متادیتای بسته: `/usr/local/share/raylimit/`
- دایرکتوری پیکربندی: `/etc/raylimit/`

### بررسی نصب

```bash
command -v raylimit
raylimit version
raylimit --help
```

## مسیر ۲: اجرا از سورس

برای اجرای مستقیم از سورس:

```bash
go run ./cmd/raylimit --help
```

برای ساخت باینری محلی:

```bash
make build
./bin/raylimit --help
```

برای اجرای فرمان‌های واقعی از بیلد محلی:

```bash
sudo ./bin/raylimit discover
sudo ./bin/raylimit inspect --pid 1234
```

## کدام مسیر برای شما مناسب است؟

مسیر بستهٔ انتشار مناسب است اگر:

- می‌خواهید `raylimit` در مسیر استاندارد سیستم در دسترس باشد
- می‌خواهید روی میزبان واقعی یا شبه‌تولیدی کار کنید

مسیر سورس مناسب است اگر:

- در حال ارزیابی یا توسعه هستید
- نمی‌خواهید نصب سیستمی انجام دهید
- می‌خواهید سریع با `go run` یا `./bin/raylimit` کار کنید

برای ادامه، از نوار کناری به بخش فرمان‌های رایج و استفادهٔ عملی بروید.
